# Introduction to SOC (Security Operations Center) – TryHackMe

## Overview

This write-up summarizes my learning from the *Introduction to SOC* room on TryHackMe. It covers the role of a Security Operations Center, responsibilities of SOC analysts, and basic security monitoring concepts.

---

## Key Concepts

### What is a SOC?

* A centralized team that monitors and analyzes security events
* Responsible for detecting, investigating, and responding to threats

---

### SOC Analyst Roles

* Monitor alerts and logs
* Investigate suspicious activity
* Respond to security incidents
* Escalate issues when needed

---

### Security Monitoring

* Continuous monitoring of systems and networks
* Identifying unusual or malicious behavior

---

### Logs and Data Sources

* Windows Event Logs
* Network logs
* Application logs
* Security tools (EDR, SIEM)

---

### SIEM (Security Information and Event Management)

* Collects and analyzes logs from multiple sources
* Helps detect threats and generate alerts

---

### Incident Response

* Process of handling security incidents
* Steps include:

  * Detection
  * Analysis
  * Containment
  * Recovery

---

## Practical Commands / Activities

* Reviewed different types of logs (Windows, network)
* Explored how alerts are generated in a SOC
* Understood basic investigation workflow

---

## What I Learned

* SOC is critical for real-time threat detection
* Logs are the primary data source for analysis
* SIEM tools help centralize and correlate data
* Analysts must think logically and investigate step by step

---

## SOC Perspective

* Identifying patterns in logs helps detect attacks
* Multiple failed login attempts may indicate brute-force attacks
* Understanding logs is essential for incident investigation
* SIEM tools are key for monitoring large-scale environments
