# What Cloud Computing Actually Is

One of the main things I understood is that cloud computing allows users and companies to access computing resources over the internet instead of relying only on their own physical machines.

These resources can include:

- Servers
- Storage
- Databases
- Networking
- Applications
- Virtual machines

Instead of buying and maintaining expensive hardware, companies can rent resources from cloud providers whenever needed.

### Real-world example

Platforms like:

- Netflix
- Spotify
- Instagram
- Google Drive

all rely heavily on cloud infrastructure to serve millions of users worldwide.

# Cloud Service Models

I also learned that cloud computing is divided into different service models depending on how much control users need.

### IaaS (Infrastructure as a Service)

Provides virtual machines, networking, and storage.

Users manage:

- Operating systems
- Applications
- Configurations

### PaaS (Platform as a Service)

Provides platforms where developers can build and deploy applications without managing low-level infrastructure.

### SaaS (Software as a Service)

Provides ready-to-use software over the internet.

Examples:

- Gmail
- Google Docs
- Microsoft 365

### Real-world understanding

This section helped me understand that different businesses use different cloud models depending on their needs.

# Virtual Machines and Cloud Instances

One of the most useful parts of the room was learning about cloud instances and virtual machines.

A cloud instance is basically a virtual machine running on cloud infrastructure.

Instead of using a physical computer directly, users can deploy virtual systems remotely through cloud providers.

In the practical section, I deployed and interacted with a cloud instance.

This helped me understand:

- How virtual machines are created in the cloud
- How remote systems are managed
- How cloud resources can be accessed over the internet

# Scalability and Resource Management

This room also explained scalability properly.

Cloud platforms allow systems to scale depending on demand.

### Example

If a website suddenly receives huge traffic:

- More server resources can be added automatically
- Traffic gets distributed efficiently
- Performance remains stable

Without scalability, websites may crash during high traffic situations.

### Real-world example

Large online platforms during sales or events rely heavily on cloud scalability.

# Cloud Computing and Cybersecurity

This room also made me realize why cloud security is important.

Since cloud systems store huge amounts of data and services online, security becomes extremely important.

Some security concerns include:

- Misconfigured cloud storage
- Weak access control
- Data leaks
- Poor authentication

Understanding cloud fundamentals is useful in cybersecurity because many companies now host their infrastructure in cloud environments.
